# 로또

> 로또 구입 금액을 입력하고 당첨 번호와 보너스 번호를 입력하면, 당첨 통계와 수익률을 출력해주는 로또 발매기

## 로또 규칙

- 기본적인 로또의 추첨 방식은 한국 로또(로또 6/45) 규칙을 따른다.
- 총 5개의 순위가 있으며, 각 순위에 해당하는 기본 당첨금은 다음과 같다.
    - 1순위(20억): 6개의 번호 모두 일치
    - 2순위(3000만원): 5개 번호 일치 + 나머지 1개가 2등 보너스볼 번호 일치
    - 3순위(150만원): 5개 번호 일치
    - 4순위(5만원): 4개 번호 일치
    - 5순위(5천원): 3개 번호 일치
- 수익률은 `(전체 당첨금) / (복권 구매 비용) * 100`으로 계산하고, 소수점 둘째 자리에서 반올림 한다.

### 실행 방법

1. 해당 Github 저장소를 클론한다.
```bash
git clone https://github.com/bhoon716/kotlin-lotto-8
```
2. IDE를 이용하여 해당 프로젝트를 연다.

3. Java 버전을 확인하고, gradle 테스트를 진행하여 결과가 아래와 같이 나오는지 확인한다.
```bash
java -version    # 버전이 21인지 확인

./gradlew clean test    # Mac
gradlew.bat clean test  # Windows
```
```bash
BUILD SUCCESSFUL in 0s
```

4. `Application.kt`의 main 함수를 실행한다.

## 실행 과정

1. 로또 구입 금액 입력
2. 구입 금액에 맞춰서 로또 발행 및 출력
3. 당첨 번호 입력(쉼표(,) 기준으로 분리)
4. 보너스 번호 입력
5. 구입한 로또 당첨 확인
6. 총 당첨 통계 출력
7. 총 수익률 출력

- [ ] [출력] `구입금액을 입력해 주세요.` 문구 출력
- [ ] [입력] 구입금액 입력
- [ ] [출력] `n개를 구매했습니다.` 문구 출력
- [x] [기능] n개의 로또 랜덤 발행
    - [x] [기능] 랜덤으로 중복없는 숫자 6자리 생성
    - [x] [기능] 숫자 6개 로또 생성
- [ ] [출력] 발행한 로또 모두 출력(각 로또 번호는 오름차순 정렬)
- [ ] [출력] `당첨 번호를 입력해 주세요.` 문구 출력
- [ ] [입력] 당첨 번호 입력
- [ ] [출력] `보너스 번호를 입력해 주세요.` 문구 출력
- [ ] [입력] 보너스 번호 입력
- [x] [기능] 당첨 로또 생성
- [x] [기능] 구매한 로또들과 당첨 로또 비교
    - [x] [기능] 로또끼리 비교 후 일치하는 숫자 개수 반환
    - [x] [기능] 특정 숫자가 로또 번호에 포함되는지 반환
    - [x] [기능] 구매한 로또들의 당첨 순위 계산
- [x] [기능] 구매한 로또 당첨 통계 계산
- [ ] [출력] 당첨 통계 출력
- [x] [기능] 수익률 계산 (수익률은 소수점 둘째 자리에서 반올림)
- [ ] [출력] 수익률 출력

## 예외 케이스

> 사용자가 잘못된 값을 입력할 경우, `IllegalArugmentException` 발생시킨 후 애플리케이션 종료

- [x] 잘못된 구입 금액 입력
    - [x] 0원 이하
    - [x] 10만원을 초과
    - [x] 자연수가 아닌 숫자나 문자
    - [x] 로또 구매 후, 잔돈(나머지) 발생
- [ ] 잘못된 로또 발행
    - [x] 6개가 아닌 당첨 번호
    - [ ] 로또 범위를 벗어난 숫자
    - [ ] 로또 번호 내 중복 존재
- [ ] 잘못된 당첨 번호 입력
    - [ ] 6개가 아닌 당첨 번호
    - [ ] 로또 범위를 벗어난 숫자
    - [ ] 정수가 아닌 숫자, 문자, 공백
    - [ ] 당첨 번호 내 중복 존재
- [ ] 잘못된 보너스 번호 입력
    - [x] 당첨 번호와 중복
    - [x] 범위 벗어남
    - [ ] 정수가 아닌 숫자, 문자, 공백

## 추가한 기능 요구 사항

> 기능 요구 사항에 기재되지 않은 내용은 스스로 판단하여 구현한다.

- 한 회차당 최대 10만원 구매제한
- 0원 입력 불가

## 주요 프로그래밍 요구사항

- build.gradle 파일은 변경할 수 없으며, 제공된 라이브러리 이외의 외부 라이브러리는 사용하지 않는다.
- indent(인덴트, 들여쓰기) depth를 3이 넘지 않도록 구현한다. 2까지만 허용한다.
- 3항 연산자를 쓰지 않는다.
- JUnit 5와 AssertJ를 이용하여 정리한 기능 목록이 정상적으로 작동하는지 테스트 코드로 확인한다.
- 함수(또는 메서드)의 길이가 15라인을 넘어가지 않도록 구현한다.
    - 함수(또는 메서드)가 한 가지 일만 잘 하도록 구현한다.
- else, switch/case를 사용하지 않는다.
- 구현한 기능에 대한 단위 테스트를 작성한다. (UI 로직 제외)
- 제공된 `Lotto` 클래스를 사용하여 구현해야 한다
- Enum을 적용한다
- `Lotto`에 `numbers` 이외의 필드(인스턴스 변수)를 추가할 수 없다
- `numbers` 의 접근 제어자인 `private` 은 변경할 수 없다
- `Lotto` 의 패키지를 변경할 수 있다
